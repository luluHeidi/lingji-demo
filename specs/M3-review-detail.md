## A. 页面定位

审核人员在详情页中查看作品信息、流程进度，按当前环节填写审核意见并提交。

### B2. 流程进度条

根据 task_type 动态展示不同的审核链路：

| task_type | 步骤 | Rule 依据 |
|-----------|------|-----------|
| authorization_review | 初审 → 复审 → 终审 → 运营授权评估 → 权益确认 | §二 2.3 三级审核 |
| risk_review_distribution | 初审 → 法务判定 | §二 2.3 风险审查 |
| risk_review_defect | 初审 → 法务判定 | §二 2.3 |
| defect_reaudit | 初审 → 复审 → 终审 → 运营授权评估 → 权益确认 | §二 2.3 权益瑕疵重审 |
| manual_recovery | 初审 → 法务复核 → 执行回收 | §五 5.1 |
| manual_reauth | 外循环发起 → 品类运营回填 → 补函确认 | §五 5.2 |

### B3. 审核表单（按环节）

**初审表单**（一期已有，不改）：
- 初审结论（下拉：权利可用/权利瑕疵）
- 可授权平台（多选：不限/抖音/快手/小红书/微博/微信视频号/其他/未约定）
- 备注（文本）

**复审表单**（一期已有，不改）：
- 复审结论（下拉）
- 全部复审字段
- 备注

**终审表单**（**二期变更**）：
- 终审结论
- **选择授权项目**：多选复选框列表，从 running 状态项目中列出（suspended/expired 不出现）
- 每个项目选项显示：项目名称 + 合作方 + 状态
- 授权起止日期从项目配置自动读取，可人工微调

## C. 交互清单

| 序号 | 触发操作 | 预期结果 | Rule 依据 |
|------|---------|---------|-----------|
| 1 | 点击面包屑「授权管理」 | 返回审查池列表，恢复一级面包屑 | 通用导航 |
| 2 | 在当前环节填写表单后点击「提交」 | 提交审核意见，进度前进到下一环节（mock） | §七 7.1 |
| 3 | 终审环节勾选授权项目 | 勾选的项目高亮，授权日期自动读取 | §三 3.4 终审交互 |
| 4 | 终审提交 | 对勾选项目创建授权关联（mock），审查单变为「已完成」 | §三 3.4 |

---

## D. 边界条件

| 序号 | 条件描述 | 预期表现 |
|------|---------|---------|
| 1 | 审查单已完成 | 所有表单只读，无提交按钮 |
| 2 | 终审时无 running 状态项目 | 项目选择列表为空，提示"暂无可授权项目" |
| 3 | 风险审查类型（2步流程） | 进度条只显示 2 步，无终审环节 |
| 4 | 手动回收/重新授权类型 | 进度条显示对应 3 步，表单字段适配 |
