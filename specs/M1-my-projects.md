# M1 执行规格 — 一级页面 Tab1：我的项目

> **模块编号**：M1
> **Rule 章节**：§三 3.4「Tab1 我的项目」+ §二（项目实体定义）+ §四（项目状态机）
> **依赖**：无
> **版本**：v1.0

---

## A. 页面定位

运营管理者在「我的项目」中查看所有授权项目、筛选定位、点击进入项目详情。

---

## B. 字段清单

### B1. 项目卡片展示字段

| 序号 | 字段名 | 数据来源 | 数据类型 | 示例值 | 备注 |
|------|--------|---------|---------|-------|------|
| 1 | 项目名称 | project.name | string | 快手二创授权2026 | 卡片主标题 |
| 2 | 权利类型 | project.rights_type | enum | 二创权+剪辑权 | 定义维度一，用标签展示 |
| 3 | 审核方式 | project.audit_mode | enum | 多轮滚动审核 | 定义维度二，用标签展示 |
| 4 | 项目状态 | project.status | enum | running | 状态 badge |
| 5 | 合作方 | project.partner_name | string | 快手 | 可为空（内部项目） |
| 6 | 品类范围 | project.category_scope | array | ["电视剧","电影","动漫"] | 展示为逗号分隔文本 |
| 7 | 授权时间窗口 | project.auth_window_start ~ auth_window_end | date | 2026-01-01 ~ 2026-12-31 | 起止日期 |
| 8 | 已授权数 | project.authorized_count | number | 3500 | 统计数字（从关联表聚合） |
| 9 | 已回收数 | project.recovered_count | number | 12 | 统计数字（从关联表聚合） |

### B2. 筛选区字段

| 序号 | 筛选项 | 数据来源 | 选项值 | Rule 依据 |
|------|--------|---------|-------|-----------|
| 1 | 权利类型 | §二 2.2 枚举 | 全部/二创权+剪辑权/信网权+转授权/信网权+播控权/改编权/IP使用权/综合权益/AIGC权益 | §三 3.4 |
| 2 | 审核方式 | §二 2.3 枚举 | 全部/多轮滚动审核/单次审批/免审核 | §三 3.4 |
| 3 | 状态 | §四 状态机 | 全部/配置中/运行中/已暂停/已到期/已归档 | §三 3.4 |

> 注意：`deleted` 状态不出现在筛选项和列表中（Rule §四：deleted 项目数据不可导出/不可见）。

### B3. 项目状态枚举映射

| status 值 | 中文标签 | badge 颜色 |
|-----------|---------|:----------:|
| configuring | 配置中 | 灰色 |
| running | 运行中 | 绿色 |
| suspended | 已暂停 | 橙色 |
| expired | 已到期 | 红色 |
| archived | 已归档 | 灰色（浅） |

### B4. 权利类型枚举映射

| rights_type 值 | 中文标签 |
|----------------|---------|
| secondary_creation | 二创权+剪辑权 |
| distribution | 信网权+转授权 |
| broadcast | 信网权+播控权 |
| adaptation | 改编权 |
| ip_usage | IP使用权 |
| comprehensive | 综合权益 |
| aigc | AIGC权益 |

### B5. 审核方式枚举映射

| audit_mode 值 | 中文标签 |
|---------------|---------|
| rolling | 多轮滚动审核 |
| one_time | 单次审批 |
| no_audit | 免审核 |

## C. 交互清单

| 序号 | 触发操作 | 预期结果 | Rule 依据 |
|------|---------|---------|-----------|
| 1 | 点击项目卡片 | 进入该项目的详情页（二级页面），隐藏一级页面所有元素 | §三 3.4 二级页面 |
| 2 | 点击「创建项目」按钮 | 弹出创建项目弹窗/流程（3步：选权利类型→选审核方式→填配置项） | §三 3.4 创建项目流程 |
| 3 | 创建 Step1：选择权利类型 | 展示 7 种权利类型选项（§二 2.2 枚举），选择后进入 Step2 | §三 3.4 |
| 4 | 创建 Step2：选择审核方式 | 展示 3 种审核方式（§二 2.3），系统根据权利类型推荐默认值，选择后进入 Step3 | §三 3.4 |
| 5 | 创建 Step3：填写配置项 | 表单：合作方名称/合同编号/联系人/品类范围(多选)/授权时间窗口(起止)/合同安全线(仅 rolling) | §二 2.4 配置项 |
| 6 | 创建 Step3 确认提交 | 创建项目（状态=configuring），返回列表并刷新 | §四 状态机 |
| 7 | 筛选「权利类型」下拉变更 | 列表按选中的权利类型过滤 | §三 3.4 |
| 8 | 筛选「审核方式」下拉变更 | 列表按选中的审核方式过滤 | §三 3.4 |
| 9 | 筛选「状态」下拉变更 | 列表按选中的状态过滤 | §三 3.4 |

---

## D. 边界条件

| 序号 | 条件描述 | 预期表现 |
|------|---------|---------|
| 1 | 项目列表为空（无任何项目） | 显示空态提示"暂无项目，点击「创建项目」开始" |
| 2 | 筛选组合后无匹配项目 | 显示"暂无匹配项目" |
| 3 | 合作方为空的项目（内部项目） | 合作方字段显示"-" |
| 4 | configuring 状态项目 | 卡片可点击进入详情，展示配置中 badge |
| 5 | deleted 状态项目 | 不出现在列表中 |
| 6 | 创建项目时未填必填项 | 提交按钮置灰或点击后提示"请填写必填项" |
| 7 | 项目进入详情后返回 | 恢复一级页面所有元素显示，列表状态不变 |

## E. 验收检查表

### 字段验证
- [ ] 卡片字段1「项目名称」：来自 project.name，渲染为卡片主标题
- [ ] 卡片字段2「权利类型」：来自 project.rights_type，按 B4 枚举映射为中文标签
- [ ] 卡片字段3「审核方式」：来自 project.audit_mode，按 B5 枚举映射为中文标签
- [ ] 卡片字段4「项目状态」：来自 project.status，按 B3 枚举映射为带颜色 badge
- [ ] 卡片字段5「合作方」：来自 project.partner_name，为空时显示"-"
- [ ] 卡片字段6「品类范围」：来自 project.category_scope 数组，逗号分隔展示
- [ ] 卡片字段7「授权时间」：来自 auth_window_start ~ auth_window_end
- [ ] 卡片字段8「已授权数」：来自 project.authorized_count
- [ ] 卡片字段9「已回收数」：来自 project.recovered_count
- [ ] 筛选项1「权利类型」：7个选项+「全部」，来自 §二 2.2
- [ ] 筛选项2「审核方式」：3个选项+「全部」，来自 §二 2.3
- [ ] 筛选项3「状态」：5个选项+「全部」（不含 deleted），来自 §四

### 交互验证
- [ ] 点击卡片 → 进入项目详情，一级页面元素全部隐藏
- [ ] 点击「创建项目」→ 弹出3步创建流程
- [ ] Step1 展示 7 种权利类型选项
- [ ] Step2 展示 3 种审核方式，有默认推荐
- [ ] Step3 表单含合作方/合同编号/联系人/品类/时间窗口/安全线(仅rolling)
- [ ] Step3 确认 → 创建项目（configuring），返回列表
- [ ] 3 个筛选器各自独立生效
- [ ] 3 个筛选器组合生效

### 边界验证
- [ ] 0 个项目 → 空态提示
- [ ] 筛选组合无结果 → "暂无匹配项目"
- [ ] 合作方为空 → 显示"-"
- [ ] deleted 项目不出现在列表
- [ ] 详情页返回后一级页面恢复

### 反向检查
- [ ] 代码中无任何规格未定义的自创字段
- [ ] 代码中无任何规格未定义的自创交互
- [ ] 代码中无任何一期逻辑的未声明变更

## G. 一期变更（本模块）

| 序号 | 变更内容 | 一期现状 | 二期改为 | Rule 依据 | 必要性 | 影响范围 |
|------|---------|---------|---------|-----------|-------|---------|
| 1 | 项目卡片字段 | 一期有 audit_count/pending_count | 去掉审查单/待审统计，只保留已授权/已回收 | §三 3.2（项目只管理审查完成后的授权结果） | 审查在中心池做，项目级无待审概念 | 项目卡片渲染函数 |
| 2 | 项目筛选 | 一期按场景/状态筛选 | 改为权利类型/审核方式/状态三维筛选 | §三 3.4 Tab1 | 二期项目由正交维度定义 | 筛选区 HTML + 筛选逻辑函数 |
