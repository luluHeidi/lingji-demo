## 一线操作员视角

### 阻塞问题

**[OPS-BLOCK-1] 已授权作品行缺少「发起回收」操作按钮**

- **位置**：`main.js:987`
- **spec 要求**：D4 "已授权作品行 → 操作列显示「发起回收」按钮"；C3 "选中已授权作品 → 点击「发起手动回收」"
- **实际表现**：当 `recovery_status` 非 `recovered` 时，操作列渲染为空字符串 `''`。已授权作品行没有任何操作按钮。
- **运营影响**：运营发现问题作品后**无法在片单内发起回收**，必须走线下流程，回收效率归零。这是 Tab A 三大核心场景之一（对账/查询/**回收**），缺失此入口等于模块功能不完整。
- **修复建议**：非回收状态且 `copyright_status` 以 `authorized` 开头的行，操作列显示「发起回收」按钮，点击后打开 M8 回收弹窗或 toast 提示 mock 流程。

**[OPS-BLOCK-2] CID 筛选器缺失（9 个筛选器只实现了 8 个）**

- **位置**：`main.js:878-933`（`renderTabAContent` 筛选区 HTML）
- **spec 要求**：B3 筛选序号 2 "CID — 文本输入"
- **实际表现**：`tabAFilters` 对象中定义了 `cid` 键（`main.js:839`），但筛选区 UI 没有渲染 CID 输入框，`filterTabAItems()` 中也没有 CID 过滤逻辑，`bindTabAFilterEvents()` 中也没有绑定 CID 事件。
- **运营影响**：版权运营日常用 CID（主CID）做跨系统关联查询是高频操作（CID 是琅琊阁、思远等系统的通用索引）。缺少此筛选器，运营只能用关键字搜索模糊匹配，效率显著降低。
- **修复建议**：在筛选区增加 CID 文本输入框，在 `filterTabAItems()` 中增加 CID 匹配逻辑（mock 数据中目前无 `cid` 字段，需同步补充）。

### 建议问题

**[OPS-WARN-1] 品类映射表 PLAY_CATEGORY_MAP 存在编码错误，导致多条 mock 数据品类显示错误**

- **位置**：`main.js:2425-2431`
- **问题**：映射表将 code `'3'` 映射为"电视剧"、`'4'` 映射为"电影"，但 mock 数据中：
  - "斗罗大陆3"（动漫）`play_category:"4"` → 显示为"电影"
  - "创造营2026"（综艺）`play_category:"3"` → 显示为"电视剧"
  - "蓝色星球中国篇"（纪录片）`play_category:"5"` → 显示为"综艺"
  - "超级小熊猫"（少儿）`play_category:"6"` → 显示为"纪录片"
- **运营影响**：运营按品类筛选时，筛出的结果与实际不符，影响对账准确性。虽然这在真实数据上取决于后端映射，但 demo 验收中品类展示错误会误导评审判断。
- **建议**：确认编码标准后修正映射表，或直接修改 mock 数据的 `play_category` 值使其与映射表一致。

**[OPS-WARN-2] 列标题"IP等级"与 spec 不一致，spec 定义为"热度"（新热-授权口径）**

- **位置**：`main.js:956`（表头）、`main.js:908-909`（筛选区 label）
- **spec 要求**：B1 列 13 为"热度"，B3 筛选项 7 为"新热-授权口径"
- **实际表现**：表头和筛选区均使用"IP等级"
- **运营影响**：运营内部统一使用"热度"口径，demo 中使用"IP等级"可能造成概念混淆。不影响功能但影响运营认知一致性。

**[OPS-WARN-3] 异步导出提示未实现（>5000 条场景）**

- **位置**：`main.js:1037-1081`（`openTabAExportModal`）
- **spec 要求**：B4 注释 "异步导出提示（>5000条）也用 toast 模拟"
- **实际表现**：导出弹窗选择模板后直接 toast "导出成功（mock）"，未根据数据量判断是否显示异步导出提示。
- **运营影响**：Demo 层面不影响功能验收，但如果后续测试用大数据量 mock，需要补充此逻辑以覆盖运营真实的大批量导出场景。

## 运营决策层视角（参考）

### 观察和建议

1. **信息层级基本合理**：作品名称、版权ID、品类等核心识别信息在前几列，授权日期和状态紧随其后，回收相关信息在末尾。符合运营"先定位作品 → 再看授权状态 → 最后看异常"的查阅习惯。

2. **品类运营主管的日常查询可满足**：筛选器覆盖了品类、日期范围、热度、信网权状态等主要维度，支持组合筛选。品类运营主管可以快速按"电视剧+S级+生效中"定位高优作品。

3. **建议后续增加排序功能**：当前表格无列排序，运营如需按"授权结束日期"升序查看即将到期作品，只能通过日期范围筛选间接实现。建议后续迭代增加列头点击排序。

4. **导出能力满足对账和核查两大场景**：两套导出模板分别覆盖对外对账（18字段）和内部核查（37+字段），运营高频使用的两个导出场景均有入口。
