# 端口管理脚本使用说明

## 📦 脚本列表

| 脚本文件 | 用途 | 使用场景 |
|---------|------|----------|
| `check-port.sh` | 检查端口可用性 | 启动前检查 |
| `clean-ports.sh` | 清理旧端口 | 端口被占用时 |
| `start-dev.sh` | 启动开发服务器 | 日常开发 |

```bash
# 已添加执行权限，可以直接使用

```


## 🔧 常用操作

### 检查端口是否可用

```bash
./scripts/check-port.sh 5173

# 如果可用：
# ✅ 端口 5173 可用

# 如果被占用：
# ⚠️  端口 5173 已被占用
# 占用详情：[显示进程信息]
# 建议操作：[清理命令]
```

### 清理灵机系统旧端口

```bash
./scripts/clean-ports.sh

# 会检查 8080, 8081, 8082 端口
# 交互式确认是否停止每个占用进程
```

### 手动启动（指定端口）

```bash
# Python HTTP 服务器
python3 -m http.server 5173

# npx serve
npx serve -l 5173

# 访问: http://localhost:5173
```

## ⚡ 快捷命令

### 一键检查灵机系统端口范围

```bash
for port in 5173 5174 5175 5176; do \
  if lsof -Pi :$port -sTCP:LISTEN -t >/dev/null 2>&1; then \
    echo "❌ $port 被占用"; \
  else \
    echo "✅ $port 可用"; \
  fi; \
done
```

### 一键清理所有旧端口

```bash
for port in 8080 8081 8082; do \
  lsof -ti:$port | xargs kill -9 2>/dev/null && echo "✓ 端口 $port 已清理"; \
done
```

### 查看当前所有监听端口

```bash
lsof -iTCP -sTCP:LISTEN -P -n | grep -E ':(5173|5174|5175|5176|3000)'
```

## 🐛 常见问题

### Q: 脚本提示权限不足？

```bash
# 重新添加执行权限
chmod +x scripts/*.sh
```

### Q: 端口被占用但找不到进程？

```bash
# 尝试强制清理（可能需要 sudo）
sudo lsof -ti:5173 | xargs sudo kill -9
```

### Q: 所有端口都被占用？

```bash
# 1. 先清理旧端口
./scripts/clean-ports.sh

# 2. 检查是否有僵尸进程
lsof -iTCP -sTCP:LISTEN -P

# 3. 重启终端或电脑
```

### Q: 启动后浏览器无法访问？

1. 确认服务器正在运行：`lsof -i:5173`
2. 检查防火墙设置
3. 尝试访问：`http://127.0.0.1:5173`

## 📝 最佳实践

1. **启动前检查** - 使用 `check-port.sh` 确认端口可用
2. **统一端口** - 优先使用 5173，备用 5174-5176
3. **及时停止** - 开发完成后用 Ctrl+C 停止服务器
4. **定期清理** - 每周运行一次 `clean-ports.sh`
5. **记录使用** - 更新 `端口管理方案.md` 中的端口状态表
