## 📐 架构设计

### 核心原则

1. **独立性** - 每个模块独立开发，互不影响
2. **稳定性** - 开发某模块时，其他模块代码保持不变
3. **一致性** - 所有模块共享设计规范和通用组件
4. **可扩展性** - 模块可独立升级和替换

## 🛠️ 模块开发流程

### Step 1: 准备阶段

1. **选择模块**: 从 `MODULE_REGISTRY.json` 中选择待开发模块
2. **确认ID**: 使用注册表中的模块ID
3. **创建目录**: 在 `modules/` 下创建模块目录

```bash
cd modules/
mkdir rights-overview
cd rights-overview
```

### Step 2: 创建模块文件

#### 2.1 创建 `index.html`（模块页面）

```html
<!-- modules/rights-overview/index.html -->
<div class="module-container" data-module="module-rights-overview">
    <div class="module-header">
        <h1 class="module-title">商业化权益大盘</h1>
        <div class="module-actions">
            <!-- 操作按钮 -->
        </div>
    </div>
    
    <div class="module-content">
        <!-- 模块主体内容 -->
    </div>
</div>
```

#### 2.2 创建 `style.css`（模块样式）

```css
/* modules/rights-overview/style.css */

/* 模块容器 */
.module-container[data-module="module-rights-overview"] {
    padding: 24px;
}

/* 模块标题 */
.module-header {
    display: flex;
    justify-content: space-between;
    align-items: center;
    margin-bottom: 24px;
}

.module-title {
    font-size: 24px;
    font-weight: 600;
    color: var(--text-primary);
}

/* 模块内容 */
.module-content {
    /* 使用CSS变量确保一致性 */
}
```

#### 2.3 创建 `main.js`（模块逻辑）

```javascript
// modules/rights-overview/main.js

class RightsOverviewModule {
    constructor() {
        this.moduleId = 'module-rights-overview';
        this.data = null;
    }

    async init() {
        console.log('[RightsOverview] 模块初始化');
        await this.loadData();
        this.bindEvents();
        this.render();
    }

    async loadData() {
        // 加载模拟数据
        const response = await fetch('modules/rights-overview/data.json');
        this.data = await response.json();
    }

    bindEvents() {
        // 绑定事件
    }

    render() {
        // 渲染数据
    }
}

// 监听模块加载事件
document.addEventListener('moduleLoaded', (e) => {
    if (e.detail.moduleId === 'module-rights-overview') {
        const module = new RightsOverviewModule();
        module.init();
    }
});
```

#### 2.4 创建 `data.json`（模拟数据）

#### 2.5 创建 `config.json`（模块配置）

```json

  "version": "1.0.0",
```

在 `modules/MODULE_REGISTRY.json` 中更新模块状态：

```json

}
```

## ✅ 开发规范

### 1. 文件命名规范

- HTML: `index.html` (统一命名)
- CSS: `style.css` (统一命名)
- JS: `main.js` (统一命名)
- 数据: `data.json` (模拟数据)
- 配置: `config.json` (模块配置)

### 2. CSS规范

**必须遵守**：
- 使用 `var(--变量名)` 引用设计规范中的CSS变量
- 模块容器必须添加 `data-module="模块ID"` 属性
- 样式作用域限定在模块容器内

**推荐做法**：
```css
/* ✅ 正确：使用CSS变量 */
.module-container {
    background: var(--bg-primary);
    color: var(--text-primary);
    border-radius: var(--radius-lg);
    box-shadow: var(--shadow-sm);
}

/* ❌ 错误：硬编码颜色 */
.module-container {
    background: #ffffff;
    color: #1a1a1a;
}
```

### 3. JavaScript规范

**模块类命名**：
```javascript
// 格式：{ModuleName}Module
class RightsOverviewModule { }
class MaterialQueryModule { }
```

**模块初始化**：
```javascript
// 监听模块加载事件
document.addEventListener('moduleLoaded', (e) => {
    if (e.detail.moduleId === '你的模块ID') {
        const module = new YourModule();
        module.init();
    }
});
```

**数据加载**：
```javascript
// 使用相对路径加载模块数据
async loadData() {
    const response = await fetch('modules/your-module/data.json');
    this.data = await response.json();
}
```

### 4. HTML结构规范

**标准模板**：
```html
<div class="module-container" data-module="模块ID">
    <!-- 模块头部 -->
    <div class="module-header">
        <h1 class="module-title">模块标题</h1>
        <div class="module-actions">
            <!-- 操作按钮 -->
        </div>
    </div>
    
    <!-- 模块内容 -->
    <div class="module-content">
        <!-- 内容区域 -->
    </div>
</div>
```

## 🔒 模块隔离原则

### DO（应该做）

✅ 在模块目录内开发所有功能  
✅ 使用设计规范中的CSS变量  
✅ 模块样式加命名空间（data-module属性）  
✅ 监听 `moduleLoaded` 事件初始化模块  
✅ 使用相对路径加载模块资源  
✅ 测试前先检查其他模块是否正常  

### DON'T（不应该做）

❌ 修改主框架文件（index.html, framework.css, framework.js）  
❌ 修改其他模块的文件  
❌ 使用全局变量污染命名空间  
❌ 硬编码颜色、尺寸等设计规范值  
❌ 在模块外部添加样式  
❌ 删除或移动已完成的模块

## 📊 模块状态管理

### 状态定义

| 状态 | 说明 | 操作 |
|------|------|------|
| `pending` | 待开发 | 尚未开始 |
| `developing` | 开发中 | 正在开发 |
| `testing` | 测试中 | 开发完成，测试中 |
| `completed` | 已完成 | 测试通过 |
| `published` | 已发布 | 正式上线 |

### 状态流转

```
pending → developing → testing → completed → published
   ↓          ↓           ↓          ↓
 (可跳过)   (可回退)    (可回退)   (可回退)
```

## 💡 最佳实践

### 1. 模块解耦

```javascript
// ✅ 好的做法：模块独立
class MyModule {
    constructor() {
        this.moduleId = 'module-xxx';
    }
}

// ❌ 坏的做法：依赖其他模块
class MyModule {
    constructor() {
        this.otherModule = window.otherModule; // 耦合
    }
}
```

### 2. 数据Mock

```json
// data.json 应该结构完整，便于后续对接真实API
{
  "code": 0,
  "message": "success",
  "data": {
    // 模拟真实数据结构
  }
}
```

### 3. 错误处理

```javascript
async loadData() {
    try {
        const response = await fetch('modules/xxx/data.json');
        this.data = await response.json();
    } catch (error) {
        console.error('[Module] 数据加载失败:', error);
        this.showError('数据加载失败');
    }
}
```

## 📋 素材申用流程设计

### 业务流程

**核心流程**: `选择素材 → 填写申用信息 → 提交审批 → 审批通过 → 下载素材`

### 1. 申用入口

#### 单个素材申用
- 位置：素材卡片"查看详情"弹窗中
- 操作：点击"申用"按钮

#### 批量申用
- 位置：素材详情页底部批量操作栏
- 操作：勾选多个素材 → 点击"批量申用"按钮

### 2. 申用表单字段

```json
{
  "materialIds": ["SC20260119MX01", "SC20260119MX02"],
  "applicant": "张三",
  "department": "营销部",
  "projectName": "《长相思》第二季宣传",
  "purpose": "剧宣",
  "usage": "用于制作人物小卡和宣传海报",
  "deadline": "2026-02-20",
  "remark": "需要高清原图"
}
```

**字段说明**：
- `materialIds` (必填): 申用的素材ID列表
- `applicant` (必填): 申请人姓名
- `department` (必填): 申请部门
- `projectName` (必填): 项目名称
- `purpose` (必填): 用途（剧宣/开机/设定/营销/衍生）
- `usage` (必填): 具体使用场景描述
- `deadline` (可选): 需要使用的截止日期
- `remark` (可选): 备注说明

### 3. 申用状态流转

```
pending (待审批)
    ↓
reviewing (审批中)
    ↓
approved (审批通过) → 可下载
    ↓
rejected (审批拒绝) → 可重新申请
```
