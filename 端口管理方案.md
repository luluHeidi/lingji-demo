## 🔧 端口管理最佳实践

### 1. 端口冲突检测脚本

创建 `scripts/check-port.sh`：

```bash
#!/bin/bash

PORT=$1

if [ -z "$PORT" ]; then
    echo "❌ 请提供端口号: ./check-port.sh 5173"
    exit 1
fi

# 检查端口是否被占用
if lsof -Pi :$PORT -sTCP:LISTEN -t >/dev/null 2>&1 ; then
    echo "⚠️  端口 $PORT 已被占用"
    echo ""
    echo "占用详情："
    lsof -Pi :$PORT -sTCP:LISTEN
    echo ""
    echo "建议操作："
    echo "1. 停止占用进程: kill -9 \$(lsof -ti:$PORT)"
    echo "2. 使用其他端口: 5174, 5175, 5176"
    exit 1
else
    echo "✅ 端口 $PORT 可用"
    exit 0
fi
```

### 2. 端口清理脚本

创建 `scripts/clean-ports.sh`：

```bash
#!/bin/bash

echo "🔍 检查灵机系统占用的旧端口..."
echo ""

PORTS_TO_CHECK=(8080 8081 8082)

for PORT in "${PORTS_TO_CHECK[@]}"; do
    PID=$(lsof -ti:$PORT 2>/dev/null)
    if [ ! -z "$PID" ]; then
        PROCESS=$(ps -p $PID -o comm= 2>/dev/null)
        echo "发现端口 $PORT 被占用 (PID: $PID, 进程: $PROCESS)"
        read -p "是否停止该进程？(y/n) " -n 1 -r
        echo
        if [[ $REPLY =~ ^[Yy]$ ]]; then
            kill -9 $PID
            echo "✅ 已停止端口 $PORT 的占用进程"
        else
            echo "⏭️  跳过端口 $PORT"
        fi
        echo ""
    else
        echo "✓ 端口 $PORT 未被占用"
    fi
done

echo ""
echo "✨ 端口清理完成！"
```

### 3. 启动脚本模板

创建 `scripts/start-dev.sh`：

```bash
#!/bin/bash

# 配置
DEFAULT_PORT=5173
PROJECT_NAME="灵机系统"

echo "🚀 启动 $PROJECT_NAME 开发服务器"
echo ""

# 检查默认端口
PORT=$DEFAULT_PORT
while lsof -Pi :$PORT -sTCP:LISTEN -t >/dev/null 2>&1 ; do
    echo "⚠️  端口 $PORT 已被占用，尝试下一个端口..."
    PORT=$((PORT + 1))
    if [ $PORT -gt 5180 ]; then
        echo "❌ 所有预定端口都被占用，请手动清理"
        exit 1
    fi
done

echo "✅ 使用端口: $PORT"
echo ""

# 启动开发服务器（根据项目类型选择）
# 对于静态HTML项目
if command -v python3 &> /dev/null; then
    echo "启动 Python HTTP 服务器..."
    python3 -m http.server $PORT
elif command -v npx &> /dev/null; then
    echo "启动 npx serve..."
    npx serve -l $PORT
else
    echo "❌ 未找到可用的服务器工具"
    exit 1
fi
```

## 🛠 端口管理工具命令

### 查看所有监听端口

```bash
# 查看所有监听端口
lsof -iTCP -sTCP:LISTEN -P -n

# 查看特定端口
lsof -i:5173

# 查看特定范围端口
lsof -iTCP -sTCP:LISTEN -P | grep -E ':(5173|5174|5175)'
```

### 停止占用端口的进程

```bash
# 停止特定端口的进程
kill -9 $(lsof -ti:8080)

# 批量停止多个端口
for port in 8080 8081 8082; do
    lsof -ti:$port | xargs kill -9 2>/dev/null
done
```

### 查找进程信息

```bash
# 查看端口占用的详细信息
lsof -i:5173 | grep LISTEN

# 查看进程树
ps -ef | grep [进程ID]
```

## ⚠️ 注意事项

### 端口保护规则

1. **绝对不可动的端口**
   - `3000` - 宠爱应用专属
   - `5000` - 系统服务

2. **优先使用的端口**
   - `5173` - 灵机系统主端口
   - `5174-5176` - 灵机系统备用

3. **禁止使用的端口**
   - `80/443` - 需要管理员权限
   - `1-1024` - 系统保留端口
   - `3306/5432/6379` - 数据库常用端口

### 常见问题处理

#### Q: 端口被占用但找不到进程？
```bash
# 可能是僵尸进程，尝试强制清理
sudo lsof -ti:5173 | xargs sudo kill -9
```

#### Q: 启动服务器后预览不可见？
1. 检查防火墙设置
2. 确认浏览器访问地址正确（http://localhost:5173）
3. 检查服务器是否真的在运行：`lsof -i:5173`

#### Q: 如何避免端口冲突？
1. 使用自动端口检测脚本
2. 统一使用5173-5180范围内的端口
3. 开发完成后及时停止服务器
