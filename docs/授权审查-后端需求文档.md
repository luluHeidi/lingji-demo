## 一、业务背景概述

### 1.1 模块定位

「授权审查」是腾讯视频IP商业化管理系统中，对视频作品授权进行多级人工审核的业务模块。审核流程为三级节点顺序审核（权益初审→权益复审→运营授权评估），审核完成后作品进入「已确权」状态，可用于后续授权变现。

### 1.2 核心流程

```
作品进入审查队列（系统自动/人工添加）
    → 权益初审（结论：可用/权利瑕疵）
    → 权益复审（结论：可用/权利瑕疵，补充授权依据）
    → 运营授权评估（判断：可授权/不授权，指定平台和日期）
    → 已确权
```

### 2.3 审核状态枚举（reviewStatus）

| 状态值 | 含义 | 前端显示映射 |
|--------|------|-------------|
| `权益初审待审` | 等待节点一审核 | 待初审 |
| `权益复审待审` | 等待节点二审核 | 待复审 |
| `运营授权评估待审` | 等待节点三审核 | 待终审 |
| `已确权` | 三级审核全部完成 | 已确权 |

**状态流转规则**：`权益初审待审` → `权益复审待审` → `运营授权评估待审` → `已确权`

每次提交审核结论后，后端需将 reviewStatus 推进到下一个状态。

### 3.5 添加审查IP

**请求**：

```
POST /api/auth-review/add-item
```

**请求体**：

```json
{
  "bqid": "bq2078802",
  "contractNos": ["bq2078802-o1234567"],
  "defectReason": "初审瑕疵原因文本"
}
```

**后端处理逻辑**：

1. **校验 BQID**：
   - 在作品库中查询该 BQID 是否存在
   - 不存在 → 返回错误 `BQID not found in work library`

2. **校验合同号关联性**（如果提供了合同号）：
   - 每个合同号的前缀应与 BQID 匹配
   - 不匹配 → 返回错误 `Contract number prefix mismatch`

3. **生成新条目**：
   - 生成唯一 id（格式：`RR-UGC-MANUAL-{timestamp}`）
   - 生成 taskId（格式：`SQ` + 6位随机数字 + 2位随机小写字母）
   - taskInitTime = 当前时间
   - 从作品库获取 workName、category、cid
   - externalAuthRights = `权利瑕疵`
   - reviewStatus = `权益初审待审`
   - manualAdded = true

4. **插入数据**：
   - 添加到二创权益（ugc）分类的 items 列表
   - 更新 totalPending 计数

**响应**：

```json
{
  "code": 0,
  "message": "添加成功",
  "data": {
    "item": { /* 新创建的 RightsReviewItem */ }
  }
}
```

## 四、核心业务判断逻辑

### 4.1 任务类型派生逻辑

**后端/前端均需实现的计算逻辑**：

```python
def get_task_type(item):
    """
    根据法务复审结论和榜单类型派生任务类型
    优先级：法务复审结论 > 榜单类型
    """
    if item.externalAuthRights in ('权利瑕疵', '不可用'):
        return '风险审查'
    elif item.listType == '新热':
        return '新热确权'
    else:
        return '增补确权'
```

**规则说明**：
- `externalAuthRights` 为 `权利瑕疵` 或 `不可用` → 无论 listType 是什么，都是「风险审查」
- 仅当 externalAuthRights 为 `可用` 时，才根据 listType 进一步判断
- listType 为 `新热` → 「新热确权」
- 其他所有情况 → 「增补确权」

### 4.2 审核状态推进逻辑

```python
def advance_review_status(item, node_id):
    """
    提交审核结论后，推进 reviewStatus
    """
    status_map = {
        'node-1': '权益复审待审',
        'node-2': '运营授权评估待审',
        'node-3': '已确权'
    }
    item.reviewStatus = status_map.get(node_id, item.reviewStatus)
```

### 4.3 节点可编辑性判断

```python
def can_modify_node(nodes, node_index):
    """
    已审核节点是否可修改：
    仅当下一个节点仍为 pending 状态时允许修改
    """
    if nodes[node_index].status != 'reviewed':
        return False
    next_node = nodes[node_index + 1] if node_index + 1 < len(nodes) else None
    if next_node is None:
        return False
    return next_node.status == 'pending'
```

### 4.4 活跃节点确定逻辑

```python
def find_active_node(nodes):
    """
    找到当前活跃节点（第一个 pending 节点）
    如果全部 reviewed，返回最后一个节点
    """
    for i, node in enumerate(nodes):
        if node.status == 'pending':
            return i
    return len(nodes) - 1
```

### 4.5 Node-2 数据继承逻辑

```python
def get_node2_defaults(nodes):
    """
    Node-2（权益复审）默认继承 Node-1 的结论和瑕疵类型

    node1 = next((n for n in nodes if n.id == 'node-1'), None)

            'default_defect_type': node1.defectType if node1.conclusion == '权利瑕疵' else '',
            'show_defect_fields': node1.conclusion == '权利瑕疵'

    return {
        'default_conclusion': '',
        'default_defect_type': '',
        'show_defect_fields': False
    }
```

### 4.6 Node-3 条件字段显示逻辑

```python
def get_node3_conditions(nodes):
    """
    Node-3 条件字段显示规则
    """
    node2 = next((n for n in nodes if n.id == 'node-2'), None)
    
    # 瑕疵处置结论：仅当 node-2 为权利瑕疵且有处置方式时显示
    show_defect_disposal = (
        node2 and 
        node2.status == 'reviewed' and 
        node2.conclusion == '权利瑕疵' and 
        bool(node2.disposal)
    )
    
    # 不授权原因选项：当 node-2 结论为「可用」时，排除「无法补确认函」
    all_deny_reasons = ['时长考量', '内容考量', '无法补确认函']
    if node2 and node2.status == 'reviewed' and node2.conclusion == '可用':
        deny_reasons = [r for r in all_deny_reasons if r != '无法补确认函']
    else:
        deny_reasons = all_deny_reasons
    
    return {
        'show_defect_disposal': show_defect_disposal,
        'deny_reason_options': deny_reasons
    }
```

### 4.7 授权日期默认值计算

```python
def get_default_auth_start_date(rights_items):
    """
    授权开始日期默认值 = max(今天, 权利数据中所有 authStartDate 的最大值)
    """
    today = datetime.date.today()
    max_rights_date = today
    
    for item in rights_items:
        if item.authStartDate:
            d = parse_date(item.authStartDate)  # 格式: "2026.03.01"
            if d and d > max_rights_date:
                max_rights_date = d
    
    return max(today, max_rights_date)
```

```python
def validate_node3(data, show_defect_disposal):
    errors = []
    if not data.get('authJudge'):
        errors.append('请先选择授权判断')
        return errors
    
    # 瑕疵处置结论（条件必填）
    if show_defect_disposal and not data.get('defectDisposalConclusion'):
        errors.append('请先选择瑕疵处置结论')
    
    if data['authJudge'] == '不授权':
        if not data.get('denyReason'):
            errors.append('请先选择不授权原因')
    elif data['authJudge'] == '可授权':
        platforms = data.get('authPlatform', '').split('、')
        if not platforms or platforms == ['']:
            errors.append('请至少选择一个可授权平台')
        
        platform_dates = data.get('platformDates', {})
        for platform in platforms:
            if platform not in platform_dates:
                errors.append(f'请选择{platform}授权起止日期')
            else:
                dates = platform_dates[platform]
                if not dates.get('start'):
                    errors.append(f'请选择{platform}授权开始日期')
                if not dates.get('end'):
                    errors.append(f'请选择{platform}授权结束日期')
                if dates.get('start') and dates.get('end'):
                    if dates['end'] <= dates['start']:
                        errors.append(f'{platform}授权结束日期必须晚于开始日期')
    
    return errors
```

### 4.9 筛选查询逻辑

```python
def filter_items(items, filters):
    """
    多条件AND筛选
    """
    result = items
    
    # 审核进度筛选
    if filters.get('progress'):
        progress_map = {
            '待初审': '权益初审待审',
            '待复审': '权益复审待审',
            '待终审': '运营授权评估待审',
            '已确权': '已确权'
        }
        target_status = progress_map.get(filters['progress'])
        result = [item for item in result if item.reviewStatus == target_status]
    
    # 任务类型筛选
    if filters.get('taskType'):
        result = [item for item in result if get_task_type(item) == filters['taskType']]
    
    # 品类筛选
    if filters.get('category'):
        result = [item for item in result if item.category == filters['category']]
    
    # 法务复审结论筛选
    if filters.get('extAuth'):
        result = [item for item in result if item.externalAuthRights == filters['extAuth']]
    
    # 关键词搜索（模糊匹配 workName、alias、taskId）
    if filters.get('keyword'):
        kw = filters['keyword'].lower()
        result = [
            item for item in result
            if kw in (item.workName or '').lower()
            or kw in (item.alias or '').lower()
            or kw in (item.taskId or '').lower()
        ]
    
    return result
```

### 4.12 添加审查IP的 BQID 校验

```python
def validate_add_auth_ip(bqid, contract_nos, work_library):
    """
    校验添加审查IP的输入
    """
    errors = []
    
    # 1. BQID 必填
    if not bqid:
        errors.append('请输入版权ID')
        return errors
    
    # 2. BQID 存在性校验
    matched = next((w for w in work_library if w['bqid'] == bqid), None)
    if not matched:
        errors.append(f'版权ID "{bqid}" 在作品库中未找到')
        return errors
    
    # 3. 合同号前缀关联性校验
    if contract_nos:
        for cn in contract_nos:
            if not cn.startswith(bqid):
                errors.append(f'合同号 "{cn}" 的前缀与版权ID "{bqid}" 不匹配')
    
    return errors
```
