## 一、授权审查首页字段

### 1.1 列表表格字段（10列）

| # | 字段名称 | 字段标识 | 类型 | 必填 | 样式 | 说明 |
|---|---------|---------|------|------|------|------|
| 1 | 审核任务ID | taskId | string | ✅ | 等宽字体 | 格式：SQ+6位数字+2位字母 |
| 2 | 作品名称 | workName | string | ✅ | 截断+title | — |
| 3 | 别名 | alias | string | 否 | 空值显示`-` | — |
| 4 | 品类 | category | string | ✅ | 直接展示 | 电视剧/动漫/综艺等 |
| 5 | 版权ID | bqid | string | ✅ | 等宽字体 | — |
| 6 | 任务发起时间 | taskInitTime | string | ✅ | — | YYYY-MM-DD HH:mm:ss |
| 7 | 任务类型 | (派生) | string | — | 彩色Tag | 风险审查/新热确权/增补确权 |
| 8 | 法务复审结论 | externalAuthRights | string | ✅ | 彩色标签 | 可用/不可用/权利瑕疵 |
| 9 | 审核进度 | reviewStatus(映射) | string | ✅ | 彩色Tag | 待初审/待复审/待终审/已确权 |
| 10 | 操作 | — | — | — | 按钮 | 去审核/查看详情 |

### 1.2 筛选器字段（5项）

| # | 筛选器 | 匹配字段 | 选项 |
|---|--------|---------|------|
| 1 | 审核进度 | reviewStatus | 全部/待初审/待复审/待终审/已确权 |
| 2 | 任务类型 | (派生) | 全部/风险审查/新热确权/增补确权 |
| 3 | 品类 | category | 全部+动态品类 |
| 4 | 法务复审结论 | externalAuthRights | 全部/可用/不可用/权利瑕疵 |
| 5 | 搜索 | workName/alias/taskId | 自由文本 |

### 1.3 添加审查IP弹窗字段（4项）

| # | 字段 | 标识 | 类型 | 必填 | 控件 | 说明 |
|---|------|------|------|------|------|------|
| 1 | 版权ID | bqid | string | ✅ | 文本 | 自动匹配作品库 |
| 2 | 作品名称 | workName | string | — | 自动填充 | BQID查出后自动展示 |
| 3 | 待审查合同号 | contractNos | string | 否 | 文本 | 多条用；分隔 |
| 4 | 初审瑕疵原因 | defectReason | string | ✅ | 多行文本 | 500字限制+字数统计 |

### 1.4 派生字段计算规则

**任务类型**:
- externalAuthRights ∈ {权利瑕疵, 不可用} → 风险审查
- listType = 新热 → 新热确权
- 其他 → 增补确权

**审核进度映射**:
- 权益初审待审 → 待初审
- 权益复审待审 → 待复审
- 运营授权评估待审 → 待终审
- 已确权 → 已确权

## 三、审查详情页 - 审核节点字段

### 3.1 节点一（权益初审）

| # | 字段 | 标识 | 类型 | 必填 | 控件 | 选项/说明 |
|---|------|------|------|------|------|---------|
| 1 | 人审结论 | conclusion | string | ✅ | 单选radio | 可用, 权利瑕疵 |
| 2 | 瑕疵类型 | defectType | string | 条件✅ | 单选radio | 转授或剪辑需书面确认, 联合转授, 收益共享, 无剪辑权, 非独可转授 |
| 3 | 权益初审备注 | remark | string | 否 | 多行文本 | 1000字限制 |

**条件逻辑**: 字段2-3仅在 conclusion=权利瑕疵 时显示

**已审核只读展示字段**:

| # | 字段 | 标识 | 说明 |
|---|------|------|------|
| 1 | 审核人 | reviewer | — |
| 2 | 审核时间 | reviewTime | — |
| 3 | 人审结论 | conclusion | 颜色区分 |
| 4 | 瑕疵类型 | defectType | 仅瑕疵时 |
| 5 | 备注 | remark | 仅瑕疵时 |

### 3.2 节点二（权益复审）

| # | 字段 | 标识 | 类型 | 必填 | 控件 | 选项/说明 |
|---|------|------|------|------|------|---------|
| 1 | 人审结论 | conclusion | string | ✅ | 单选radio | 可用, 权利瑕疵（默认继承node-1） |
| 2 | 授权依据的引入权利 | authBasisRightsId | string | ✅ | 单选radio | 当前IP下所有权益rightsId |
| 3 | 瑕疵类型 | defectType | string | 条件 | 单选radio | 同node-1选项（默认继承node-1） |
| 4 | 瑕疵处置方式 | disposal | string | 条件 | 单选radio | 需补剪辑确认函, 需补权利回收函, 需补其他文件待沟通 |
| 5 | 权益复审备注 | remark | string | 否 | 多行文本 | 1000字限制 |

**条件逻辑**: 字段3-5仅在 conclusion=权利瑕疵 时显示

**已审核只读展示字段**:

| # | 字段 | 标识 |
|---|------|------|
| 1 | 审核人 | reviewer |
| 2 | 审核时间 | reviewTime |
| 3 | 人审结论 | conclusion |
| 4 | 授权依据的引入权利 | authBasisRightsId |
| 5 | 瑕疵处置方式 | disposal |
| 6 | 瑕疵原因 | defectReason |

### 3.3 节点三（运营授权评估）

| # | 字段 | 标识 | 类型 | 必填 | 控件 | 选项/说明 |
|---|------|------|------|------|------|---------|
| 1 | 授权判断 | authJudge | string | ✅ | 单选radio | 可授权, 不授权 |
| 2 | 瑕疵处置结论 | defectDisposalConclusion | string | 条件✅ | 单选radio | 可补确认函, 无法补确认函, 不补函 |
| 3 | 不授权原因 | denyReason | string | 条件✅ | 单选radio | 时长考量, 内容考量, 无法补确认函 |
| 4 | 可授权平台 | authPlatform | string | 条件✅ | 多选checkbox | 快手, 抖音 |
| 5 | {平台}授权开始日期 | platformDates.{平台}.start | date | 条件✅ | 日期选择 | min=今天 |
| 6 | {平台}授权结束日期 | platformDates.{平台}.end | date | 条件✅ | 日期选择 | min=开始日期 |

**条件逻辑**:
- 字段2: 仅当node-2结论=权利瑕疵且有处置方式时显示
- 字段3: 仅当authJudge=不授权时显示（node-2结论=可用时排除「无法补确认函」选项）
- 字段4-6: 仅当authJudge=可授权时显示，每勾选一个平台生成一组日期

**已审核只读展示字段**:

| # | 字段 | 标识 |
|---|------|------|
| 1 | 审核人 | reviewer |
| 2 | 审核时间 | reviewTime |
| 3 | 授权判断 | authJudge |
| 4 | 瑕疵处置结论 | defectDisposalConclusion |
| 5 | 不授权原因 | denyReason |
| 6 | 可授权平台 | authPlatform |
| 7 | {平台}授权起止时间 | platformDates.{平台}.display |
