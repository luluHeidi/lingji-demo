## 新规则版本列表

### V1.9.0 - M9 重新授权三级页 (2026-04-24)

**对应 M 文件**: M9-reauth-page
**模块**: auth-management（授权管理）

**核心内容**:
- 重新授权三级页完整工作流（pending → group_confirm → pending_supplement → authorized/rejected）
- 流程模块布局：已完成节点展示结论，流程中左结论右操作，全流程结束横排展示
- 3步骤操作：发起重授权（外循环运营）→ 判断是否可补函（品类运营）→ 确认补函结果（品类运营）
- Step 3 增加"经确认无法补函"分支 → rejected
- 每个操作节点可选备注
- 历史记录表横向按次数展示（8列：次数/发起时间/外循环操作人/补函结论/补函结论操作人/补函确认结果/补函确认操作人/备注）
- 操作人 = 登录用户（权限系统控制，无需下拉选择）
