## 🎯 功能特性

### 核心功能（7大系统）

1. **IP资产展示系统**：120个示例IP，三品类切换
2. **智能搜索系统**：名称模糊/ID精准/AI智能三种模式
3. **筛选与过滤系统**：下拉框筛选+关键词搜索+组合筛选
4. **素材详情展示系统**：素材卡片+筛选+批量操作
5. **素材申用确认系统**：申用表单+素材清单
6. **导航与状态管理**：面包屑+页面状态保持
7. **模块加载优化**：并行加载+缓存机制

### 技术亮点

- ✅ 模块化架构（代码隔离、职责清晰）
- ✅ 三级页面结构（列表→详情→操作）
- ✅ 状态管理（页面往返状态保持）
- ✅ 智能搜索（三种模式自动切换）
- ✅ 性能优化（并行加载、防抖、缓存）
- ✅ UI/UX优化（清透阴影、品牌色统一）

## 🔒 使用限制

### 锁定说明

本版本是**锁定版本**，意味着：

✅ **允许**：
- 查看和学习代码
- 基于快照创建新模块
- 作为参考和学习资料
- 在其他模块中调用

❌ **禁止**：
- 直接修改 `modules/material-query/` 中的代码
- 删除本版本快照
- 覆盖版本文件

### 为什么锁定？

1. **保证稳定性**：确保已完成的模块不被意外修改
2. **代码隔离**：后续开发不会影响该模块
3. **版本管理**：清晰的版本历史和回滚能力
4. **团队协作**：避免多人同时修改同一模块

## 💡 常见问题

**Q1: 这个版本能直接运行吗？**

不能。本快照只包含模块代码，需要配合主框架（`index.html`、框架CSS/JS）才能运行。建议在完整项目中测试。

**Q2: 如何将这个版本应用到我的项目？**

```bash
# 方法1：回滚主模块
cp -r versions/v0.8.0/material-query/* modules/material-query/

# 方法2：创建新模块
cp -r versions/v0.8.0/material-query modules/my-new-module
# 然后修改config.json和注册新模块
```

**Q3: 版本快照会被自动清理吗？**

不会。版本快照是永久保存的，除非手动删除。

**Q4: 我可以修改这个快照吗？**

可以，但**不建议**。版本快照应保持原样作为历史记录。如需修改，建议：
1. 复制到新目录
2. 在新目录中修改
3. 不要覆盖原快照

**Q5: 如何查看与其他版本的差异？**

```bash
# 对比v0.7.0和v0.8.0
diff -r versions/v0.7.0/ versions/v0.8.0/
```
